  Feature Brief: Drift Dataset Generation & Procurement                         
                                                                                
  Why                                                                           
                                                                                
  The agent-in-the-loop system is graded on diagnostic correctness              
(precision/recall/false-positive rate) for drift, which requires test cases     
with known injected ground truth ‚ÄĒ                                              
  something no real dataset provides. This feature delivers the drift-team's    
data layer: a parameterized synthetic generator (the primary deliverable, for   
tool unit tests and                                                             
  agent evals) plus a small set of curated real univariate series (for the demo 
 and the "reproduce the issue" step). Without labeled drift onsets, none of the 
 downstream                                                                     
  diagnostic evals are measurable, so this is foundational.                     
                                                                                
  Scope                                                                         
                                                                                
  In scope: univariate time series only; gradual drift (the abrupt/localized    
case is the changepoint team's domain). Dataset generation and procurement ‚ÄĒ    
not detection tools,                                                            
  not the agent, not forecasting.                                               
                                                                                
  Out of scope: changepoints, outliers, multivariate series, the diagnostic     
tools themselves.                                                               
                                                                                
  What ‚ÄĒ Synthetic Generator (primary)                                          
                                                                                
  A seeded, parameterized generator producing univariate series with injected,  
ground-truth-labeled drift. It must support four drift flavors, each mapping to 
 a distinct Prophet                                                             
  component (y = g(t) + s(t) + h(t) + őĶ) and thus a distinct downstream         
intervention:                                                                   
                                                                                
  1. Trend-slope drift ‚ÄĒ gradual change in trend slope (g(t)).                  
  2. Mean-level ramp ‚ÄĒ gradual shift in level (g(t)).                           
  3. Variance inflation ‚ÄĒ gradual growth in noise spread (őĶ).                   
  4. Seasonal-amplitude drift ‚ÄĒ seasonal swings grow/shrink over time, i.e.     
additive‚Üímultiplicative (s(t)).                                                 
                                                                                
  Required knobs (per the proposal's "knobs/parameters" requirement) with       
reasonable defaults:                                                            
  - drift onset (start index/date)                                              
  - drift magnitude                                                             
  - transition width (őĒt) ‚ÄĒ the gradual-vs-abrupt control; this is the shared   
parameter that distinguishes drift from a changepoint, and lets us generate     
deliberately ambiguous                                                          
  middle-ground cases                                                           
  - drift duration                                                              
  - base noise level                                                            
  - seasonality (period, base amplitude)                                        
  - random seed (reproducibility is mandatory)                                  
                                                                                
  Ground-truth labels emitted with each case: drift flavor, onset index,        
transition width, affected component ‚ÄĒ sufficient to compute detection          
precision/recall.                                                               
                                                                                
  Case composition: core eval set = one injected flavor per case (clean ground  
truth for metrics); plus a small set of combined-flavor cases as harder stretch 
 tests.                                                                         
                                                                                
  What ‚ÄĒ Real Datasets (secondary, demo only)                                   
                                                                                
  Curate ~2 real univariate series that genuinely exhibit gradual drift, for    
the demo and forecast-degradation reproduction. Candidates:                     
  - Air Passengers ‚ÄĒ growing seasonal amplitude (real-world twin of synthetic   
seasonal-amplitude drift).                                                      
  - Mauna Loa CO‚āā ‚ÄĒ slow rising level + seasonality.                            
                                                                                
  Accepted caveat: real series have no labeled drift onset, so they are         
excluded from precision/recall evals ‚ÄĒ they exist only for qualitative          
demonstration.                                                                  
                                                                                
                                                                                
  Visual confirmation (acceptance aid)                                          
                                                                                
  Each generated/curated series should be visually verifiable as actually       
containing drift via a standard overlay: series + rolling mean + rolling std,   
with the injected onset                                                         
  marked (reproducing Figure 3 of the proposal). This is the eyeball gate for   
"is this dataset good?"                                                         
                                                                                
  Success criteria                                                              
                                                                                
  - Generator produces all four flavors, fully parameterized and seeded         
(reproducible).                                                                 
  - Every synthetic case carries machine-readable ground-truth labels usable    
for precision/recall.                                                           
  - Ambiguous (mid-őĒt) cases can be generated on demand.                        
  - ~2 real demo series are loadable through the same interface.                
  - A visualization confirms drift presence for any case.
